# CS 320 Portfolio Reflection
How can I ensure that my code, program, or software is functional and secure?

Throughout this course, I learned that writing code is only one part of developing quality software. To make sure my code is functional and secure, I use unit testing to verify that each method behaves as expected and that invalid inputs are handled correctly. I also validate user input, test boundary conditions, and create both positive and negative test cases. Using JUnit helped me identify problems early and gave me confidence that the application met the required specifications before it was considered complete.

How do I interpret user needs and incorporate them into a program?

I start by carefully reviewing the software requirements and breaking them into smaller, testable pieces. During this course, I used the customer requirements to develop the Contact, Task, and Appointment services, making sure every requirement had corresponding code and unit tests. This approach helped me focus on what the customer expected while also verifying that the software behaved correctly under both normal and invalid conditions.

How do I approach designing software?

I approach software design by first understanding the requirements and then creating small, well-organized classes that each have a specific responsibility. I build the functionality step by step and write unit tests alongside the code to verify that each feature works correctly. This project reinforced the importance of testing throughout development instead of waiting until the end. Moving forward, I plan to continue using this approach because it helps reduce defects, improves maintainability, and produces software that is more reliable and easier to support over time.
